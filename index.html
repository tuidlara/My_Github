Conceitos

Git é um sistema de controle de versão distribuído que permite que múltiplos desenvolvedores trabalhem em um projeto simultaneamente, rastreando mudanças no código-fonte ao longo do tempo. Ele facilita a colaboração, o gerenciamento de versões e a recuperação de versões anteriores do código.

GitHub é uma plataforma baseada na web que utiliza o Git como sistema de controle de versão. Ele oferece uma interface amigável para hospedar repositórios Git, facilitando a colaboração entre desenvolvedores, o compartilhamento de código e a integração com outras ferramentas de desenvolvimento.

Fluxo de Git e Github

O fluxo típico de trabalho com Git e GitHub envolve:

  1. Git init: Inicializa um repositório vazio
  2. Git clone: Clona um repositório existente
  3. Git add: Adiciona arquivos ao índice para serem commitados
  4. Git commit: Salva as mudanças no repositório local
  5. Git push: Envia as mudanças para o repositório remoto no GitHub
  6. Git pull: Atualiza o repositório local com as mudanças do repositório remoto
  7. Git branch: Cria, lista ou exclui branches
  8. Git merge: Mescla branches diferentes
  9. Git stash: Guarda todas as alterações não commitadas e limpa a área de trabalho
  10. Git stash apply: Aplica o stash mais recente, mantendo ele na lista
  11. Git status: Mostra o status atual dos arquivos
  12. Git checkout nome-da-branch: Muda para outra branch
  13. Git remote add origin 'url': Conecta o repositório local a um remoto

Como resolver conflitos

Conflitos ocorrem quando fazem alterações conflitantes no mesmo arquivo ou linha de código. Para resolver conflitos no Git, siga estes passos:

  1. Identifique os arquivos em conflito usando o comando git status.
  2. Abra os arquivos em conflito e procure por marcadores de conflito (<<<<<<<, =======, >>>>>>>).
  3. Edite os arquivos para resolver os conflitos, escolhendo quais mudanças manter.
  4. Depois de resolver os conflitos, adicione os arquivos resolvidos ao índice com git add nome-do-arquivo.
  5. Finalize o merge com um commit usando git commit.
  6. Se estiver trabalhando com um repositório remoto, envie as mudanças resolvidas com git push.
